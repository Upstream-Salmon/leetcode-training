拡張性のあるロジック実装と計算量理論の基礎

## 議論のまとめ
FizzBuzzという基本的な問題を通じ、単なる条件分岐からデータ駆動型の設計への昇華プロセスを議論した。具体的には、3や5といった個別のルールを LinkedHashMap で管理し、StringBuilder を用いて動的に文字列を構築することで、将来的なルールの追加にコードの修正なしで対応できる設計を学んだ。また、Javaの文法（Map.EntryやStringBuilderのメソッド）の正確な記述と、面接で必須となる計算量分析（時間・空間）の基礎概念を整理した。

## 構成要素と学習ポイント
- 拡張性: マジックナンバー（15など）を排除し、個別の条件を組み合わせて複合条件を作る設計。
- Javaベストプラクティス: ループ内での文字列連結に StringBuilder を使用し、パフォーマンスを最適化。
- 計算量分析: 
    - 時間計算量: O(n * K) ※Kはルール数
    - 空間計算量: O(n) ※結果格納用リストに依存
- 補助空間（Auxiliary Space）: アルゴリズム自体が追加で使用するメモリと、出力用メモリの区別。
